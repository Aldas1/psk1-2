How Optimistic Locking Works

This demonstration shows how JPA's optimistic locking mechanism works:

  1. Each entity has a version field that is automatically incremented when the entity is updated.
  2. When updating an entity, JPA checks if the version number matches what's in the database.
  3. If the version numbers don't match (meaning someone else updated the record), an OptimisticLockException is thrown.
  4. The proper way to recover is to get a fresh copy of the entity and reapply your changes.
