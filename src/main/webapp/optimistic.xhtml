Explanation:

What happens with the transaction when OptimisticLockException occurs?

When an OptimisticLockException occurs, the current transaction is marked for rollback. Any changes made in that transaction are discarded and will not be committed to the database.

What happens with the EntityManager when OptimisticLockException occurs?

The EntityManager remains usable for read operations, but any attempt to commit the transaction will result in a RollbackException. To save the entity after an OptimisticLockException, you need to:

  1. Begin a new transaction
  2. Reload the entity from the database to get the latest version
  3. Reapply your changes
  4. Merge and commit
