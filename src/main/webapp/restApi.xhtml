RESTful API Endpoints

The following RESTful API endpoints are available for integration with other systems.

Faculty API

GET /api/faculties

Retrieves a list of all faculties.

GET /api/faculties/{id}

Retrieves a specific faculty by ID.

POST /api/faculties

Creates a new faculty.

// Request body example:
{
  "name": "Faculty of Science",
  "department": "Mathematics"
}
PUT /api/faculties/{id}

Updates an existing faculty.

// Request body example:
{
  "name": "Faculty of Science",
  "department": "Computer Science"
}
DELETE /api/faculties/{id}

Deletes a faculty by ID.

Course API

GET /api/courses

Retrieves a list of all courses.

GET /api/courses/{id}

Retrieves a specific course by ID.

POST /api/courses

Creates a new course.

// Request body example:
{
  "courseCode": "CS101",
  "title": "Introduction to Programming",
  "credits": 6,
  "facultyId": 1
}
PUT /api/courses/{id}

Updates an existing course.

// Request body example:
{
  "courseCode": "CS101",
  "title": "Advanced Programming",
  "credits": 8,
  "facultyId": 1
}
DELETE /api/courses/{id}

Deletes a course by ID.

Testing the API

You can test the API using curl commands from the command line:

# Get all faculties
curl -X GET http://localhost:8080/university/api/faculties

# Create a new faculty
curl -X POST http://localhost:8080/university/api/faculties \
     -H "Content-Type: application/json" \
     -d '{"name":"Faculty of Arts","department":"Music"}'

# Get a specific faculty
curl -X GET http://localhost:8080/university/api/faculties/1

# Update a faculty
curl -X PUT http://localhost:8080/university/api/faculties/1 \
     -H "Content-Type: application/json" \
     -d '{"name":"Faculty of Arts","department":"Visual Arts"}'

# Delete a faculty
curl -X DELETE http://localhost:8080/university/api/faculties/1
